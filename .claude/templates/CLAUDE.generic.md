# Claude Code 作業指針テンプレート（汎用・他プロジェクト転用可）

このファイルは特定プロジェクトに依存しない、汎用的な「トークン節約 / 作業効率化」の
指針テンプレートです。他のプロジェクトフォルダにコピーし、冒頭にそのプロジェクト固有の
概要（アーキテクチャ・ファイル構成・設定値の場所など）を追記した上で `CLAUDE.md` として
配置してください。Claude Codeはリポジトリ直下の `CLAUDE.md` を自動で読み込みます。

## 使い方

1. このファイルを対象プロジェクトの `CLAUDE.md` としてコピー
2. ファイル冒頭に、そのプロジェクト固有の「概要 / アーキテクチャ / ファイル構成 / 設定値の
   場所」セクションを追記する
3. 下記の「作業時の指針」はそのプロジェクトの実情に合わせて微調整してよい
   （例: 大きいファイルの閾値、使用言語、テストコマンド等）
4. あわせて `.claude/settings.json`（`settings.generic.json` を参照）もコピーすると、
   読み取り専用コマンドの許可プロンプトを減らせる

## 作業時の指針（トークン節約・汎用版）

- **大きいファイルを丸ごと読まない**: 数百行〜を超えるファイルは、まず `Grep` で目的の
  シンボル/キーワードを検索し、該当箇所だけを `offset`/`limit` 付きで `Read` する。
- **広範囲・横断的な調査はサブエージェントに委任する**: 「〇〇を使っている箇所を全部探して」
  のような探索は `Explore` 系のエージェントに投げ、要約結果だけを受け取る。自分でも同じ検索を
  重複して行わない。
- **読み直しを避ける**: 直前に読んだファイルや、自分がEdit/Writeした内容を確認目的で
  再度Readしない。Editツールは失敗時にエラーを返すため、成功の確認のための再読込は不要。
- **出力を絞る**: `git log` やコマンド出力は必要な範囲だけ（`head_limit` や `-n` オプション、
  `--oneline` 等）を使う。ログ全件表示など大きな出力を無条件に取得しない。
- **スコープを守る**: 依頼された修正の範囲に留め、ついでのリファクタや将来のための抽象化を
  持ち込まない。三行の重複は早すぎる抽象化より安全。
- **既知の技術的負債・運用上のクセをCLAUDE.mdに書き残す**: 「このファイルは実際には
  使われていない」「このバージョン文字列は変更のたびに更新が必要」など、踏むと時間を溶かす
  落とし穴は都度追記し、将来のセッションが同じ調査をゼロからやり直さずに済むようにする。
- **このセッションでできないこと・確認できないことは正直に書く**:
  デプロイ確認・外部サービスへの実アクセスなど、この実行環境から検証できない事項は
  「未検証」と明記し、ユーザー側での確認を依頼する。

## コミット/PRの基本方針

- 明示的に指示されない限りコミット・pushをしない
- 破壊的なgit操作（force push, reset --hard 等）は指示がない限り行わない
- PRを作成する場合はテンプレート(`.github/pull_request_template.md`等)の有無を確認し、
  あれば見出し構成を踏襲する
